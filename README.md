# 🛡️ Runtime Application Self-Protection (RASP)
### Executable Context Isolation & Memory-Space Threat Neutralization

[![Security Paradigm](https://img.shields.io/badge/Paradigm-Active%20Self%20Defense-FF2E93?style=for-the-badge)](https://www.nist.gov/publications/zero-trust-architecture)
[![Telemetry Layer](https://img.shields.io/badge/Telemetry-Kernel%20%2F%20Runtime%20Hooks-0052CC?style=for-the-badge)]()

A programmatic, production-grade implementation of a Runtime Application Self-Protection (RASP) engine. This framework shifts security boundaries directly into the application's memory space, enabling workloads to dynamically monitor executing code paths and instantly kill malicious payloads (such as Remote Code Execution and Web Shells) from within the execution runtime itself.

## 📊 WAF Bypass vs. Inside-the-Code RASP Defense

### 🔄 Scenario A: The Perimeter Bypass (Traditional WAF Limitation)
Traditional Web Application Firewalls (WAF) operate strictly at the perimeter layer. When an attacker delivers an obfuscated or nested polymorphic payload, the WAF is completely blind to the threat, allowing the execution context to become heavily compromised.
